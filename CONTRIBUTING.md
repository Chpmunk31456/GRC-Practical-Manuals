# Contributing to GRC Practical Manuals

Thank you for your interest in improving the GRC Practical Manuals collection.

This repository is intended to provide free, practical, beginner-friendly guidance in governance, risk, compliance, cybersecurity, privacy, resilience, cloud security, and related areas.

Constructive corrections, recommendations, and contributions are welcome.

## Ways to Contribute

You may contribute by:

- Reporting spelling or grammar errors
- Reporting broken links
- Identifying outdated information
- Recommending clearer explanations
- Suggesting new topics or manuals
- Recommending authoritative sources
- Improving accessibility
- Suggesting diagrams, checklists, or templates
- Reporting formatting problems
- Suggesting translations or localization improvements

## Reporting an Issue

Please use the repository’s GitHub Issues section.

Include the following information when possible:

- Manual name
- Version number
- File type
- Page or section number
- Description of the issue
- Recommended correction
- Supporting authoritative source
- Screenshot, when helpful

## Accuracy and Source Requirements

Recommendations should be supported by reliable and current sources whenever possible.

Preferred sources include:

- Official government publications
- Official regulatory authorities
- Official standards organizations
- Framework owners
- Recognized professional associations
- Current vendor documentation
- Peer-reviewed research
- Authoritative technical documentation

Please do not submit unsupported claims, copied commercial material, or confidential organizational information.

## Writing Style

Contributions should be:

- Clear
- Practical
- Respectful
- Beginner-friendly
- Professionally written
- Free from unnecessary jargon
- Accurate and responsibly sourced
- Safe, ethical, and legally appropriate

## Security and Ethics

Do not contribute instructions that encourage:

- Unauthorized system access
- Illegal activity
- Privacy violations
- Credential theft
- Malicious software
- Circumvention of security controls
- Harmful or deceptive conduct

Cybersecurity examples must emphasize authorization, safety, ethics, and legal responsibility.

## Copyright and Licensing

Only submit material that you created or have permission to share.

By contributing, you agree that your contribution may be included under the repository’s license:

**Creative Commons Attribution-NonCommercial-ShareAlike 4.0 International**

Do not submit copyrighted standards text, commercial course material, proprietary reports, confidential documents, or restricted content.

## Translations

Translations should be properly localized rather than produced as literal word-for-word translations.

Contributors should review:

- Spelling
- Grammar
- Technical terminology
- Regional language usage
- Tone
- Accessibility
- Cultural clarity

Machine-assisted translations require human review by a proficient speaker before they are presented as reviewed editions. Translation proposals should identify the language, locale, reviewer, and source edition.

## Accessibility

Review [ACCESSIBILITY.md](./ACCESSIBILITY.md) before changing document structure or producing new Word or PDF editions. Correct the source document before regenerating derivative formats.

## Pull Requests

When submitting a pull request:

1. Describe the change clearly.
2. Identify the affected manual and section.
3. Explain why the change is needed.
4. Include supporting sources when applicable.
5. Keep unrelated changes separate.
6. Confirm that links and filenames work.
7. Review spelling, grammar, formatting, and readability.

## Review Process

All contributions may be reviewed for:

- Accuracy
- Relevance
- Safety
- Ethics
- Licensing
- Readability
- Source quality
- Consistency with the repository’s educational mission

Not every recommendation will be accepted, but all respectful and constructive feedback is appreciated.

## Contact and Questions

Use GitHub Issues for:

- Corrections
- Questions
- Suggestions
- Requests for new topics
- Accessibility concerns
- Translation recommendations
